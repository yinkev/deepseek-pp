## DeepSeek++ 0.7.3

0.7.3 是项目上下文和权限收敛稳定性版本，让 DeepSeek++ 在项目任务、浏览器控制、记忆注入、自动化和长任务续跑中表现更稳，同时减少不必要的权限负担。

### 主要变化

- **项目上下文更稳**：项目指令、项目记忆和已关联 DeepSeek 对话的上下文使用更稳定，处理项目任务时更少需要重复粘贴背景。
- **Browser Control 权限更克制**：Browser Control 不再依赖额外的标签组权限；功能仍默认关闭，只有用户启用并选择目标标签页后才会提供页面文本快照和可见浏览器动作。
- **记忆和提示词链路收紧**：记忆筛选、项目记忆和提示词准备更准确，减少上下文缺失、重复或进入错误任务范围的情况。
- **自动化和长任务续跑更稳**：自动化任务状态、工具结果回传和多步续跑继续打磨，更适合定时跟踪和需要持续推进的任务。
- **流式输出更干净**：长回复里的工具调用和上下文注入处理更可靠，降低技术标记残留到正文或打断正常输出的概率。
- **回归覆盖继续补强**：新增和补强项目上下文、项目记忆作用域、浏览器控制权限、同步数据、自动化状态和流式工具文本测试，发布前继续覆盖多浏览器构建与资产校验。

中文：

- 项目上下文更稳定：项目指令、项目记忆和相关 DeepSeek 对话会更可靠地协作，减少重复粘贴背景。
- Browser Control 移除不必要的标签组权限依赖；仍然只有用户启用并选择目标标签页后才会提供文本快照和可见网页操作。
- 记忆、自动化任务和长任务续跑稳定性增强，降低上下文错位、状态错位和流式输出中断。

English:

- Project context is more reliable: project instructions, project memories, and related DeepSeek conversations work together with less repeated background pasting.
- Browser Control removes an unnecessary tab-group permission dependency; text snapshots and visible web actions are still available only after the user enables the feature and selects a target tab.
- Memory, automation tasks, and long-task continuation are more stable, reducing wrong-scope context, state mismatches, and streamed-output interruptions.
- Strengthens regression coverage for project context, Browser Control permissions, sync data, automation state, and streamed tool text.
