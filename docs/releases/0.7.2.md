## DeepSeek++ 0.7.2

0.7.2 是浏览器控制和侧边栏体验增强版本，让 DeepSeek++ 可以在用户选定的标签页中执行可见网页操作，同时把权限边界、第三方 Skill 管理和操作反馈讲得更清楚。

### 主要变化

- **新增浏览器控制**：可在侧边栏「能力 > 浏览器」中开启 Browser Control、选择目标标签页，并让模型执行导航、点击、悬停、输入、按键、等待页面内容、处理对话框和上传文件等网页动作。
- **权限边界更清晰**：Chrome / Edge 包新增浏览器控制所需权限；功能默认关闭，只有用户启用并选择目标标签页后才会提供页面文本快照和浏览器动作工具。
- **侧边栏反馈更明确**：保存项、项目资料和提示词控制补强成功、失败、后端不可用等状态提示，减少操作后不知道是否生效的情况。
- **第三方 Skill 更好管理**：第三方 Skill 按来源分组展示，并支持独立启用、停用、检查更新和同步，更适合维护较大的 Skill 库。
- **平台能力展示更准确**：能力页会区分浏览器扩展、Android WebView 和不支持平台的功能边界，避免把不可用能力显示成可用。
- **回归覆盖继续补强**：新增浏览器控制、侧边栏交互、运行时广播、平台能力和 Skill 本地化测试，发布前继续覆盖多浏览器构建与资产校验。

### 验证

- `npm pack --dry-run --workspace packages/shell-host`
- `node scripts/web-search-smoke.mjs`
- `npm run ci:quality`：覆盖 workflow、生产依赖审计、提示词冻结、编译、Vitest、中英文校验、自动化、MCP / Shell / PoW smoke、多浏览器构建、manifest policy、打包和 release 资产校验。

English:

- Browser Control stays off by default; text snapshots and browser action tools are added to new conversations only after the user enables the feature and selects a tab.
