Design rules for approval

  • No top horizontal tab strip as the main IA. It clips and makes the surface feel like an admin panel.
  • One composer pattern everywhere: plus, slash, mic, model dropdown, send.
  • Projects become list-first. Forms move behind New or Manage.
  • Settings becomes a clean option page pattern like Claude, with System access preserved.
  • Suggestions are allowed as chips. Everything else uses rows, menus, or simple panels.
  • Personalization must come from local stores or current tab context. No fake claims.
